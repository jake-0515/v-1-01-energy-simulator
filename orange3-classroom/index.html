📊 1단계: 머신러닝의 기초와 데이터 전처리

인공지능이 학습하는 원리를 배우고, 학습을 시작하기 전에 데이터를 정교하게 다듬는 방법을 상세히 알아봅시다.

🤖 1. 머신러닝(Machine Learning)이란 무엇일까요?

기존의 컴퓨터 프로그램은 사람이 일일이 규칙을 코딩해 주어야 작동했습니다. 하지만 **머신러닝(기계학습)**은 컴퓨터가 **대량의 데이터를 바탕으로 스스로 학습**하여 새로운 질문이나 상황에 대해 예측하고 판단하는 규칙을 찾아내는 인공지능 기술입니다.

🌾 데이터는 4차 산업혁명 시대의 '쌀'입니다!
머신러닝 학습에서 가장 중요한 것은 **신뢰할 수 있는 대량의 데이터를 확보**하는 것입니다.
이때 컴퓨터가 읽을 수 없는 한글문서(.hwp)나 PDF(.pdf) 포맷 대신, 데이터 분석기가 바로 열어서 행과 열을 분석할 수 있는 **HTML, XML, CSV, TAB** 같은 포맷으로 수집해야 학습에 활용하기 쉽습니다.
또한, 이미지나 오디오, 영상 같은 모든 형태의 데이터는 컴퓨터가 수학적 계산을 처리할 수 있도록 반드시 **수치형 데이터(숫자)**로 변환하여 입력해 주어야 합니다.

📂 2. 정형 데이터 vs 비정형 데이터

인공지능이 수집하는 모든 데이터는 구조가 잡혀있는지 여부에 따라 크게 두 가지로 분류할 수 있습니다.

📊 정형 데이터 (Structured Data)

* **정의**: 일정한 규칙에 맞춰 행과 열이 정해진 표(Table) 형태로 깔끔하게 구조화된 데이터입니다.
* **예시**: 데이터베이스 테이블, 엑셀 파일, 붓꽃(Iris) 통계치, 학생 성적부.
* **특징**: 수식 연산 및 데이터 처리가 매우 간단하며, 가명처리(개인정보 가리기) 난이도가 비교적 낮습니다.

🖼️ 비정형 데이터 (Unstructured Data)

* **정의**: 정해진 규격이나 일정한 형태가 없어 구조화되지 않은 채 존재하는 날것의 데이터입니다.
* **예시**: 강아지/고양이 사진, 음성 녹음, 메신저 대화 기록, 유튜브 동영상, 블로그 글.
* **특징**: 사전 정의된 구조가 없어 그대로는 인공지능 분석이 불가능하여 가공이 필수적이며, 처리 기술과 가명처리가 매우 복잡하고 다양합니다.

🌐 3. 데이터 수집의 보물창고: 공공 데이터 포털

국가 기관이나 국제 기구들이 보유하고 있는 다양한 공공 데이터를 누구나 무료로 다운로드받아 인공지능 분석과 앱 개발에 활용할 수 있도록 개방해 놓은 웹사이트입니다.

🗺️ 대표적인 공공 데이터 플랫폼:

  • 🇰🇷 **대한민국 공공데이터포털**: data.go.kr (공공 안전, 교통, 기상 통계 등 자료의 요람)
  • 🇺🇸 **미국 공공데이터**: data.gov
  • 🇪🇺 **유럽 연합 데이터**: data.europa.eu
  • 🏦 **국제 기구 데이터**: 세계은행(World Bank), 세계보건기구(WHO), OECD, NASA 포털.

⚖️ 4. 데이터 범위 재조정: 표준화와 정규화

여러 변수를 함께 학습시킬 때 각각의 수치 범위가 다르면 특정 변수에만 가중치가 쏠리는 문제가 생깁니다. 이를 방지하기 위해 데이터 범위를 통일하는 **전처리(Rescaling)**를 수행합니다.

📏 정규화 (Normalization)

데이터들의 최솟값을 0, 최댓값을 1로 고정하여 모든 수치를 **0 ~ 1 범위 안**으로 재조정합니다.

x_new = (x - Min) / (Max - Min)
🎯 표준화 (Standardization)

데이터의 평균으로부터 얼마나 멀리 떨어져 있는지를 계산해 **평균이 0, 표준편차가 1**이 되도록 조정합니다.

x_new = (x - Mean) / StdDev
🏫 고교 1학년 시험 점수로 보는 시각적 전처리 비교

다섯 명 학생들의 수학 점수(최고 90점, 최저 45점)가 표준화 및 정규화를 통해 어떻게 공평하게 환산되는지 슬라이더 옵션을 바꿔 실시간 확인해 보세요.

원시 데이터: 가공되지 않은 학생들의 시험 원점수 (만점 기준 100점)

🧠 5. 학습(Fitting)의 원리와 지도/비지도 학습

인공지능은 학습 방식과 정답의 유무에 따라 세 가지 유형으로 나뉩니다.

🟢 지도학습 (Supervised Learning)

학습에 사용되는 데이터에 **정답(Label, 라벨링)**이 지정되어 있는 경우입니다. 인공지능에게 정답 족보를 주고 공부를 시켜, 나중에 새로운 질문에 대한 정확한 정답을 분류 및 예측합니다. (예: 회귀분석, 개와 고양이 이미지 분류)

🔵 비지도학습 (Unsupervised Learning)

학습용 데이터에 **정답이 없는** 경우입니다. 인공지능이 스스로 데이터의 유사한 성질과 패턴을 스스로 분석하여 비슷한 무리끼리 묶어주는 학습을 진행합니다. (예: 군집화, 추천 알고리즘)

🟡 강화학습 (Reinforcement Learning)

정답 대신 자신의 행동에 따른 **보상(Reward)**을 줌으로써, 보상을 극대화하는 최적의 행동 방법을 컴퓨터가 시행착오를 통해 스스로 터득하게 만드는 기법입니다. (예: 바둑 AI 알파고, 자율주행)

🎯 적합시키기 (Fitting)와 변수 탐색

머신러닝에서 **적합시키기(Fitting)**란 수집한 데이터들의 패턴을 가장 잘 반영하는 수학적 **추세선(방정식)**이나 **분류 경계선**을 찾아내는 과정입니다. 이 선을 찾는 것은 데이터의 영향력을 결정하는 **가중치(Weight)**와 **보정값(Bias)**을 미세하게 조정하여 맞추는 행위입니다.

🐟 예시 A: 붕어빵 판매량 예측 (회귀분석) 겨울철 기온과 붕어빵 판매 데이터를 그래프에 점으로 찍고(산점도), 이 분포를 관통하는 직선($y = ax + b$)을 수학적으로 피팅해 둡니다. 이 선이 생기면 "기온이 영상 5도일 때 붕어빵은 대략 50개 팔릴 것이다"라고 판매량을 명확하게 예측할 수 있습니다.
🐈 예시 B: 고양이와 개 분류 (분류 경계선) 고양이 사진 100장(라벨 0)과 강아지 사진 100장(라벨 1)의 특징값들을 분포 공간에 펼친 뒤, 두 집단을 가장 고르게 쪼개고 구별해 줄 수 있는 최적의 **분류 경계선(Decision Boundary)**을 긋는 과정이 바로 피팅(Fitting)입니다.

⚖️ 6. 훈련 데이터(Train)와 테스트 데이터(Test) 분할

우리가 모은 데이터를 모델 학습에만 전부 사용해 버리면 인공지능이 문제를 제대로 푸는지 공정하게 시험할 수 없습니다. 모의고사 문제를 다 외워버렸기 때문이죠!

📊
* **훈련 데이터 (Train Data - 70%)**: 인공지능이 유형을 분석하고 학습(Fitting)을 하는 주재료로 쓰입니다.
* **테스트 데이터 (Test Data - 30%)**: 학습 과정에서는 절대 보여주지 않고 숨겨둔 뒤, 인공지능의 학습이 완료된 후 실제 정답값과 인공지능의 예측값 간의 차이(오차)를 채점하여 성능과 정확도를 정밀 테스트하는 데 활용합니다.

🤖 이미지 임베딩 (Image Embedding) 이란?
컴퓨터는 고양이나 강아지 사진을 픽셀 그대로 분석하여 지도학습하기 어렵습니다. 구글의 합성곱신경망(CNN) 기반 모델이 이미지에서 털의 재질, 눈과 귀의 모양, 주둥이 비율 등을 분석하여 수천 개의 숫자 정보(특성 벡터, Feature Vector)로 가공해주는 기법입니다.
