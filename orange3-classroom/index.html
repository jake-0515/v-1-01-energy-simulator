📊 1단계: 머신러닝의 기초와 데이터 전처리

인공지능이 스스로 학습하는 기계학습(머신러닝)의 원리와 데이터를 가공하는 기초 상식을 알아봅시다.

🤖 1. 머신러닝(기계학습)이란 무엇일까요?

사람이 컴퓨터에게 일일이 규칙을 코딩해 지시하는 대신, 컴퓨터가 **과거의 대량의 데이터**를 분석하여 스스로 규칙이나 예측 공식을 찾아내게 만드는 인공지능 기술입니다.

📂 2. 데이터의 종류: 정형 vs 비정형

컴퓨터가 학습하는 데이터는 형태에 따라 크게 2가지로 나뉩니다.

  • 정형 데이터 (Structured Data): 엑셀 표나 데이터베이스 테이블처럼 행과 열이 정해진 깔끔한 수치 데이터 (예: 기온 통계, 붓꽃 측정치)
  • 비정형 데이터 (Unstructured Data): 그림, 음성, 텍스트처럼 규격화되지 않은 일상의 자료 (예: 개/고양이 사진, 동영상)

🧠 3. 머신러닝의 3대 학습 방식

🎯 A. 지도학습 (Supervised Learning)

과거 데이터에 **원인(Feature)**과 **정답(Target)**을 함께 족보처럼 주어 학습시키는 방식입니다. (예: 기온으로 판매량 예측하기, 붓꽃 품종 분류)

🔵 B. 비지도학습 (Unsupervised Learning)

**정답(라벨)이 없는 상태**에서 컴퓨터 스스로 데이터의 특징을 비교하여 비슷한 무리끼리 묶어주는 학습입니다.
💡 과일 가게 비유: 이름표(정답)가 없는 사과, 배, 바나나를 과일 장수 AI가 오직 색깔과 형태(Feature)의 유사점만 보고 자동으로 분류하는 것과 같습니다! (대표 위젯: K-Means 군집화)

🟢 C. 강화학습 (Reinforcement Learning)

행동에 대한 잘잘못에 따라 **보상(Reward)**을 줌으로써, 바둑판이나 운전 중 최고의 점수를 얻을 수 있는 최적의 경로를 스스로 시행착오를 통해 깨우치게 하는 방식입니다. (예: 알파고)

⚖️ 4. 훈련(Train) 데이터 vs 테스트(Test) 데이터

가지고 있는 데이터를 전부 공부에만 쓰면 시험 능력을 평가할 수 없습니다. 오렌지3에서는 보통 데이터를 **훈련용 70%(공부용)**와 **테스트용 30%(모의고사용)**로 쪼개어 인공지능이 처음 보는 데이터도 잘 맞히는지 정밀 평가합니다.

🎯 3단계: Orange3 지도학습 실습 교실

오렌지3를 활용해 수치 예측(회귀)과 품종 판독(분류)을 진행하는 두 가지 수업 시나리오를 직접 체험해 봅시다.

📈 레몬에이드 예측 모델 시나리오

* **원인 변수 (Feature)**: 기온 (온도가 높을수록 더 목이 마르겠죠?)
* **결과 변수 (Target)**: 레몬에이드 판매량 (**수치형 양적 데이터**)
* **핵심 알고리즘**: **Linear Regression (선형 회귀)**

🎬 1차시 수업 현장 속으로
교사: "여러분, 만약 여러분이 레몬에이드 카페 사장님인데, 내일 주스를 몇 잔이나 팔지 미리 알 수 있다면 어떨까요?"
학생: "재료를 딱 맞게 준비할 수 있어서 버리는 게 없을 것 같아요!"
교사: "맞아요! 원인이 되는 변수를 Feature, 결과 변수를 Target이라고 불러요. 숫자를 예측하는 머신러닝 기법을 '회귀(Regression)'라고 해요."
교사: "Linear Regression 위젯을 데이터와 연결하면 인공지능이 스스로 공식을 학습합니다."
학생: "우와! 온도가 24도일 때 48개가 팔릴 거라고 정답을 딱 예측해 줘요!"
교사: "과거 데이터를 분석해서 규칙을 찾아 미래를 맞힌 것입니다. 회귀 마스터 완료!"
🌸 붓꽃 품종 알아맞히기 시나리오

* **원인 변수 (Feature)**: 꽃잎 길이, 꽃잎 너비 (수치형 측정치)
* **결과 변수 (Target)**: 붓꽃 품종 (Setosa / Versicolor / Virginica - **범주형 데이터**)
* **알고리즘 시합**: **Logistic Regression (로지스틱 회귀)** vs **Decision Tree (의사결정나무)**

💡 알아두면 똑똑해지는 분류 핵심 개념!

📈 로지스틱 회귀 (Logistic Regression): "일직선으로 편 가르기!"

데이터 평면 위에 자를 대고 일직선(또는 사선)을 슥 그어서 분류하는 모델입니다. 그래서 시뮬레이터 차트에서 경계선이 비스듬한 대각선 모양으로 그려져 분류 영역이 나뉩니다.

🌳 의사결정나무 (Decision Tree): "스무고개 퀴즈 놀이!"

"길이가 2.45 이하인가?", "너비가 1.75 이하인가?" 같은 질문을 차례대로 던져서 나뭇가지처럼 분류해 나가는 알고리즘입니다. 질문이 딱딱 끊기기 때문에 경계선이 바둑판 격자(네모 박스) 모양으로 계단식 분할되는 특징을 가집니다.

🎯 CA (Classification Accuracy): "인공지능의 모의고사 시험 점수!"

분류 정확도라고 부르며, 인공지능이 문제를 풀었을 때 정답을 맞힌 비율입니다. 예를 들어 CA가 0.967(96.7%)이라면, 100개 시험 문제 중 96.7점을 받았다는 뜻으로 점수가 높을수록 더 공부를 잘한 인공지능 모델입니다!

🎬 2차시 수업 현장 속으로
교사: "결과가 숫자가 아니라 '이름'이나 '종류'일 때 사용하는 머신러닝 기법을 '분류(Classification)'라고 불러요."
학생: "붓꽃 데이터에는 Target 자리에 꽃 이름들이 들어있네요!"
교사: "맞아요. 이번엔 여러 분류 알고리즘에게 시합을 붙여서 누가 더 똑똑한지 알아봅시다."
교사: "Logistic Regression과 Tree 위젯을 Test and Score에 연결하면 모의고사 정확도(CA) 점수가 나옵니다."
학생: "선생님, 제 화면에는 Logistic Regression이 점수가 조금 더 높게 나와요!"
교사: "아주 잘 찾았어요! 인공지능이 알아낸 예측 결과를 가지고 우리 인간은 더 창의적인 의사결정에 집중하면 됩니다."

🖼️ 4단계: 비정형 데이터와 이미지 분류

숫자가 아닌 사진이나 텍스트를 인공지능은 어떻게 이해하고 분류해 내는지 학습합니다.

⚙️ 1. 이미지 임베딩(Image Embedding)이란?

고양이나 강아지 사진을 컴퓨터는 그대로 계산할 수 없습니다. 그래서 인공신경망(CNN) 모델을 거쳐 이미지 내의 특징점(눈의 간격, 귀의 뾰족함, 주둥이 길이 등)을 수많은 대표 숫자 벡터로 변환하여 수치 데이터로 가공해 주어야 머신러닝 분류 알고리즘이 동작할 수 있습니다.

💡 비정형 데이터 가명처리와 특징 추출의 난이도:

텍스트나 사진은 개인정보(얼굴, 주민번호 등)가 무작위로 포함될 수 있어 가명처리(개인정보 가리기) 난이도가 정형 데이터보다 훨씬 복잡하고 가공 시 많은 가중치 분석 연산이 요구됩니다.
